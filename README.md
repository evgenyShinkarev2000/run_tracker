Для локального запуска в web используется nginx для установки правильных заголовков для sqlite3.wasm и drift_worker.js с проксированием, должно работать из коробки, настройки в ./web_dev_config.yaml  ./nginx ./lib/Data/AppDatabase.dart

drift для сохранения бд может использовать IndexedDB или origin private file system. При изменении схемы бд нужно удалить старую бд или использовать миграции. flutter может сохранять данные в ./dart_tool/chrome-device, из-за чего очистка всех данных в браузере может не сработать, т.к. при запуске chrome в дебаг создается новая временная директория, куда копируются файлы с ./dart_tool/chrome-device. IndexedDB может не отображаться https://issues.chromium.org/issues/406991275, для просмотра и редактироваия opfs нужно установить расширение https://chromewebstore.google.com/detail/opfs-explorer/acndjpgkpaclldomagafnognkcgjignd. Чтобы это сделать с debug режимом, можно создать в chrome чистый профиль, установить расширение, скопировать папку с профилем C:\Users\user\AppData\Local\Google\Chrome\User Data\Default (вместо Default может быть другое название, после копирования переименовать в Default), в файле {flutter sdk path}flutter\packages\flutter_tools\lib\src\web\chrome.dart удалить флаги --bwsi --disable-extensions, в переменной final Directory userDataDir указать путь до скопированной папки с профилем Default (сам Default в путь не включать), удалить файл {flutter sdk path}\flutter\bin\cache\flutter_tools.stamp, затем запустить приложение или выполнить flutter doctor для обновления flutter_tools.snapshot.
